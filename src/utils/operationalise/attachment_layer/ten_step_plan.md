# Ten-Step Plan: Local Positive Curvature Operationalisation

Status: active programme plan.

The optimal next move is to strengthen the idea before broadening the artifact set. The idea is not an Evidence schema. The idea is local positive curvature admissibility:

```text
bare physical surface -> refusal
declared local positive curvature object -> admissible route
```

## 1. Freeze The Admissibility Principle

Write and maintain the rule that a physical hook becomes attachable only after a finite local positive curvature object is declared.

Acceptance criterion: every new object must say its curvature source, coordinate, locality, positivity, sensor/observer route, and refused routes.

## 2. Stress-Test Information Curvature

Use RC decay and Beer-Lambert to test whether information curvature behaves like a legitimate local positive curvature object.

Required tests:

- positivity;
- reparameterization covariance;
- monotone noise scaling;
- refusal under missing noise or zero sensitivity.

Current status: implemented in `local_curvature_stress_tests.py` for RC decay and Beer-Lambert.

## 3. Add Negative Controls

Do not count a proof as successful if the same logic would admit nonsense.

Required refusals:

- no declared noise model;
- zero derivative/sensitivity;
- singular Fisher object unless support is declared;
- hidden-load call without ceiling.

## 4. Promote Only After Stress Tests

Only after the proofs survive stress tests should we define a minimal evidence/Fisher attachment schema.

Acceptance criterion: schema must be a recording form for an already-understood admissibility route, not the source of the route.

## 5. Stabilize The Next Storage-Curvature Cards

Add direct static cards only where the old layer is unquestionably valid:

- `linear_elastic_bar`
- `capacitor_charge_coordinate`

Acceptance criterion: exact derivation, units, sensor surface, controls, and no hidden-load call unless a ceiling is declared.

Current status: implemented and promoted to `good_v0_storage` sign-off. These close the obvious direct storage-curvature openings before the next PDF digestion pass.

## 6. Test Finite Modal Curvature

Before creating a wave card, prove the finite mode route on a minimal fixed-end string model.

Acceptance criterion: boundary, basis, truncation, mass metric, stiffness matrix, and positivity are all explicit.

## 7. Treat Ideal Gas As Boundary Proof

Do not make a broad thermodynamics card.

Acceptance criterion: gas law alone remains refusal; a conditional isothermal Helmholtz local Hessian may be recorded only with ensemble, potential, coordinate, and operating point.

## 8. Define Schema Promotion Criteria

A schema can be promoted only when it has at least two successful instances from different ontological routes or a very clear single-route purpose.

Acceptance criterion: the schema must preserve refusal states and not silently fill missing physics.

## 9. Keep The Validator As The Spine

Every artifact must be machine-checkable where possible.

Acceptance criterion: cards, sweeps, failure probes, admissibility proofs, and stress tests validate together.

## 10. Report Fork Decisions

After the stress tests, decide whether the next best fork is:

- evidence/Fisher schema;
- finite modal schema;
- direct static storage cards;
- thermodynamic boundary proof;
- defer route as true gap.

Acceptance criterion: no fork proceeds because it is interesting; it proceeds because it passed the local admissibility test.
