# Declared Local Positive Contact v0

Status: scientific synthesis memo for the operationalisation layer.

This is not a new theorem and not a new module kernel. It is the current contact doctrine extracted from the cards, refusals, modal probes, and empirical probes.

## Claim

The attachment layer is becoming a disciplined interface for this question:

```text
What finite declared positive object can the module touch here?
```

The central lesson is not that every physical formula is a module object. The lesson is that physical surfaces become module-facing only through declared local positive contact.

The current contact pattern is:

```text
surface law / dataset / sensor relation
-> declaration package
-> finite local positive object
-> observer or sensor route
-> licensed module call or refusal
```

## Why This Is Compelling

A single successful card can be dismissed as a clever encoding. The stronger signal is that success and refusal are now lining up under the same rule.

The layer predicts, before fitting or running:

- springs and LC systems should enter through stored quadratic curvature;
- a full pendulum should refuse global quadratic attachment, while a small-angle pendulum should admit local quadratic attachment;
- a raw wave equation should refuse finite attachment until boundary, basis, truncation, and observer map are declared;
- a point sensor over modes should refuse hidden load until a ceiling or reference is declared;
- an RC voltage trace should emit information curvature for `tau`, not storage geometry or capacitance without `R`;
- Beer-Lambert spectra should emit concentration/calibration precision, not thermodynamic concentration geometry or hidden load.

That is the beginning of a predictive contact grammar rather than an example list.

## The Gold Quartet

The current best compact demonstration set is:

1. `mass_spring_coupled_observe_one` and `coupled_lc_resonators_observe_one_node`

   These show cross-substrate convergence: different physical matter, same observer-facing hidden-renormalisation pattern under partial observation and a declared visible-block ceiling.

2. `string_fixed_end_three_mode`, `string_point_sensor_modal_observer`, and `string_modal_hidden_load_with_declared_ceiling`

   These show the finite modal discipline: raw continuum language becomes attachable only after boundary, basis, truncation, sensor row, and then a declared reference ceiling if hidden load is requested.

3. `rc_decay_q7_probe`

   This shows dynamic data becoming information curvature: a voltage trace becomes local Fisher precision for `tau` only after declaring `V0`, positive `tau`, sample times, and residual-estimated voltage noise. It still refuses capacitance without `R`.

4. `beer_lambert_uvvis_probe`

   This shows a sensor-law dataset becoming analysis-function curvature: spectra and standards become a local concentration estimate and precision only after declaring wavelength, calibration standards, residual noise, and calibration span. It refuses molar absorptivity without path length.

This quartet is more persuasive than a larger inventory because it tests four distinct contact routes while preserving adjacent refusals.

## What Counts As Good Operational Contact

A contact object is good at v0 only if it declares:

- physical surface or dataset;
- finite coordinates or parameters;
- units and coordinate scale;
- regime, operating point, support, basis, or truncation;
- sensor or observer surface;
- control variables;
- curvature provenance;
- finite positive object;
- allowed module route;
- boundary and failure mode;
- adjacent refusal;
- validation result.

For empirical probes, it must also declare the uncertainty source:

| Uncertainty Source | Meaning | Current Examples |
| --- | --- | --- |
| `instrument_declared` | instrument noise/resolution is supplied externally | not yet present |
| `replicate_estimated` | variation comes from declared repeated measurements | calibration proof packets |
| `residual_estimated` | noise scale is estimated from fitted residuals | `beer_lambert_uvvis_probe`, `rc_decay_q7_probe` |
| `assumed_toy` | fixed illustrative noise used for proof only | local curvature proof packets |

These are not equivalent and should not be collapsed.

## What The Module Can Now Touch

The broader module already becomes powerful when an intelligent agent supplies the right object. The operationalisation layer is creating the object handoff.

Current handoff types:

```text
H, C -> visible precision
H, C, T -> visible precision + hidden load after ceiling dominance
K, M, C -> finite modal observer readout
measurement equation + covariance/Jacobian -> local precision record
observation equation + noise/sensitivities -> Fisher precision record
calibration function + validity range -> analysis-function precision record
refusal record -> no theorem-local call
```

The deepest emerging principle is:

```text
The same local-positive interface can be reached through different physical routes, but the licensed claim is route-dependent.
```

Energy curvature, modal curvature, and Fisher curvature can all produce finite positive objects. They do not license the same interpretations.

## Current Weaknesses

1. Apparatus realism is still thin.

   The first empirical probes use residual-estimated noise. That is acceptable for first contact, but not enough for strong external validation.

2. Thermodynamics is underdeveloped.

   The ideal gas refusal is good, but the positive thermodynamic route still needs ensemble, potential, coordinate, and fluctuation declarations.

3. Acoustic/modal externalisation is pending.

   The string modal route is strong. The natural external analogue is an acoustic finite-mode observer with pressure or displacement sensor conventions.

4. Process control is a proofed gate, not yet a full working layer.

   The first check-standard proof now distinguishes pass, drift refusal, and missing-baseline refusal. It still needs real apparatus histories before empirical probes can be trusted as live measurement processes.

5. Route promotion criteria need to remain strict.

   More examples should not dilute the doctrine. Every new success must bring a nearby refusal.

## Next Scientific Moves

1. Turn the gold quartet into the first external-facing demonstration.

   It should be readable without accepting the full theory: a physical system produces a declared finite positive object; the module call or refusal is then obvious.

2. Upgrade empirical uncertainty provenance.

   Seek at least one dataset with instrument-declared uncertainty or clean replicate structure, so the layer can distinguish residual-estimated precision from apparatus-declared precision.

3. Externalise the acoustic/modal observer route.

   The closed-tube pressure observer now tests that the string route survives a different wave substrate. The next step is apparatus realism: microphone calibration, damping, drive response, or an external frequency-response dataset.

4. Keep thermodynamics as the next deep theoretical test.

   Do not force it through the current runner. Start with a refusal-plus-conditional local Hessian or Fisher object.

5. Extend process-control gates to real apparatus histories.

   The current proof should remain gate-only. The next step is not a theorem-local call; it is a real check-standard dataset with bias, repeatability, and drift declarations.

## Operational Rule

Do not ask whether a formula or dataset is interesting first.

Ask:

```text
What is the route provenance?
What finite positive object is declared?
What observer or sensor exposes it?
Which module surfaces are licensed?
Which neighbouring claim is refused?
```

If those questions cannot be answered, the correct v0 result is refusal or deferment.
