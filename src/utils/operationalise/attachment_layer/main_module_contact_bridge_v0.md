# Main Module Contact Bridge v0

Status: wrap-ready synthesis.

This file states how the operationalise work relates back to the main observer-geometry module.

The main module is powerful when it is given the right finite object. The operationalise layer is now the discipline for deciding whether a physical surface, formula, sensor law, or dataset has actually produced such an object.

```text
physical surface
-> declaration package
-> finite positive object, gate, or refusal
-> licensed module surface or no-call record
```

## What Has Been Achieved

The phase should be wrapped as a v0 attachment grammar, not as a complete physical ontology.

The achievement is not that there are now more cards. The achievement is that route provenance, licensed claims, and refusals now line up under one contact discipline.

## Module Handoffs

| Handoff | Main Module Surface | Input Contract | Examples | Licensed Claim |
| --- | --- | --- | --- | --- |
| `visible_precision_direct` | `visible_precision` | finite SPD `H` and observer `C` | springs, LC, elastic bar, capacitor, metric, string modal, acoustic pressure observer | visible or sensor-channel precision in declared coordinates |
| `hidden_load_with_declared_ceiling` | `hidden_load` | finite SPD `H`, observer `C`, ceiling/reference `T`, and `T >= Phi` check | coupled spring, coupled LC/RLC, string modal reference ceiling | hidden-load readout relative to declared ceiling/reference |
| `local_positive_record` | record only | measurement/observation equation, likelihood, uncertainty, or residual-estimated noise | RC decay, Beer-Lambert, GUM/NIST proof packets, calibration function | covariance, Fisher, or analysis-function precision record |

| `principled_refusal` | no call | explicit missing declaration or route mismatch | full pendulum, ideal gas law, raw acoustic impedance, raw wave formula, Lorentz metric diagnostic | no module call |

## Main Lesson

The common object is not simply quadratic energy.

The common object is:

```text
declared local positive contact
```

Energy curvature, finite modal curvature, metric curvature, information curvature, and calibration curvature can all produce finite positive objects. They do not license the same interpretation. The route by which the object is produced must travel with the object.

## What Is Now Good

- Storage and modal cards give runnable module inputs.
- Hidden-load cards require an explicit ceiling/reference and executable dominance check.
- Empirical probes show that data can produce information curvature without pretending to be storage geometry.
- The process-control gate proves a pre-module refusal layer.
- Refusals are part of the scientific result, not leftovers.

## What Is Still Weak

- instrument-declared uncertainty is still absent.
- thermodynamic positive contact is still only a candidate route.
- Acoustic/modal contact is clean mathematically but not yet apparatus-real.
- Dynamic response and damping remain outside the static card runner.
- Local-positive records are not yet consumed directly by a main-module API surface.

## Wrap Recommendation

Wrap this phase as `attachment_layer_v0`: a validated contact grammar for moving from physical formula sheets and small datasets to declared module-facing objects.

Do not broaden by adding more formula cards during wrap.

The next phase should pick:

- one apparatus-grade measurement route with real uncertainty provenance;
- one thermodynamic refusal-plus-conditional-positive route;
- one API decision about how record-only Fisher/covariance objects should enter the main module.
